# NetDoctor 实机试用免环境方案

如果本机准备 Visual Studio / Windows SDK 环境比较麻烦，推荐用 GitHub Actions 云端 Windows 编译。

## 方案结论

你本机不需要安装 Visual Studio。

你只需要：

1. 把项目上传到 GitHub 仓库。
2. 打开仓库的 Actions。
3. 运行 `Build NetDoctor DLL`。
4. 下载生成的 `NetDoctor-x64.dll` 或 `NetDoctor-Win32.dll`。
5. 重命名为 `NetDoctor.dll`，放进 TrafficMonitor 的 `plugins` 目录。

## 操作步骤

### 1. 上传项目到 GitHub

新建一个 GitHub 仓库，例如：

```text
NetDoctor
```

然后在项目目录执行：

```bash
git remote add origin https://github.com/你的用户名/NetDoctor.git
git push -u origin master
```

如果你的默认分支是 `main`，也可以：

```bash
git branch -M main
git push -u origin main
```

### 2. 进入 Actions 页面

打开 GitHub 仓库页面：

```text
Actions -> Build NetDoctor DLL -> Run workflow
```

它会自动用 `windows-latest` 环境构建：

- `Release | x64`
- `Release | Win32`

### 3. 下载 DLL

构建完成后，进入本次 workflow run，在页面底部找到 Artifacts：

```text
NetDoctor-x64-Release
NetDoctor-Win32-Release
```

下载对应版本。

### 4. 判断用 x64 还是 Win32

TrafficMonitor 是 64 位，就用：

```text
NetDoctor-x64.dll
```

TrafficMonitor 是 32 位，就用：

```text
NetDoctor-Win32.dll
```

一般优先试 x64。

### 5. 安装到 TrafficMonitor

把下载到的 DLL 重命名为：

```text
NetDoctor.dll
```

放到：

```text
TrafficMonitor.exe 所在目录/plugins/NetDoctor.dll
```

然后完全退出并重启 TrafficMonitor。

## 如果 Actions 失败

优先看失败日志里是否有这些问题：

- 找不到 Windows SDK
- 找不到 MSBuild
- 平台名写错，比如 `x86` 和 `Win32` 混用
- TrafficMonitor 插件接口头文件不兼容

当前项目已经内置了 GitHub Actions 配置：

```text
.github/workflows/windows-build.yml
```

正常情况下，推到 GitHub 后就可以直接构建。
